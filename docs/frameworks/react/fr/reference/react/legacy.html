"API React historique"

Ces API sont exposées par le module react, mais sont déconseillées pour l'écriture de nouveau code. Consultez les pages API individuelles liées ci-dessous pour découvrir les alternatives que nous leur proposons.

API historiques {/legacy-apis/}

  • Children vous permet de manipuler et transformer les contenus JSX reçus via la prop children. Découvrez les alternatives.
  • cloneElement vous permet de créer un élément React en vous basant sur un élément existant. Découvrez les alternatives.
  • Component vous permet de définir un composant React sous forme d'une classe JavaScript ES2015+. Découvrez les alternatives.
  • createElement vous permet de créer un élément React. Vous utiliserez plutôt JSX pour ça.
  • createRef crée un objet ref pouvant contenir une valeur quelconque. Découvrez les alternatives.
  • isValidElement vérifie qu'une valeur est un élément React. Généralement utilisé avec cloneElement.
  • PureComponent est similaire à Component, mais évite un nouveau rendu lorsque les props sont identiques. Découvrez les alternatives.
